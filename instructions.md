# AWS Service Study Guide - Standard Instructions

Use these instructions when requesting AWS service study guides:

---

## Instructions Template

```
You are an expert tutor preparing candidates for the AWS Cloud Practitioner exam.

Create a comprehensive study guide for [SERVICE NAME] following these requirements:

CONSTRAINTS:
- Maximum 1000 lines total
- Up to 5 use cases only
- Use Digital Financial Bank as the business context
- Include Mermaid diagrams for visual concepts
- Use tables for comparisons and data organization
- No repetition or redundant explanations
- Focus on exam-relevant content

STRUCTURE (Line Budget):
1. Service Overview (50-75 lines)
   - Definition and classification
   - Key differentiators
   - Simple architecture diagram

2. Core Concepts (100-150 lines)
   - 3-5 essential concepts
   - Brief explanations with examples
   - Comparison tables where applicable

3. Key Features (200-250 lines)
   - Top 5-7 features
   - Concise descriptions
   - Feature comparison tables
   - Minimal code examples (only if essential)

4. Architecture Patterns (100-150 lines)
   - 2-3 common patterns with diagrams
   - Integration points with other AWS services

5. Real-World Use Cases (200-250 lines)
   - EXACTLY 5 use cases (40-50 lines each)
   - Cover different features and scenarios
   - Digital Financial Bank context
   - Include business outcomes

6. Integration & Deployment (75-100 lines)
   - API access methods
   - Security essentials
   - Pricing summary table

7. Exam Tips & Key Takeaways (150-200 lines)
   - Must-know facts
   - 5-6 common exam scenarios
   - Service comparison table
   - Quick reference cheat sheet

CONTENT PRIORITIES:
✅ Exam-relevant information
✅ Service comparisons with similar AWS services
✅ Common exam scenarios and patterns
✅ Digital Financial Bank use cases
✅ Tables for quick reference
✅ Mermaid diagrams for clarity

AVOID:
❌ Excessive code examples
❌ Repetitive explanations
❌ Too many architecture diagrams
❌ Detailed API documentation
❌ More than 5 use cases
❌ Overly detailed workflows

Generate the study guide now for: [SERVICE NAME]
```

---

## Quick Usage

**For next service, simply say:**

```
Follow the AWS Service Study Guide instructions in instructions.md 
and create a guide for [Amazon Polly / Amazon Translate / etc.]
```

---

## Validation Checklist

Before finalizing any guide, verify:
- [ ] Total lines ≤ 1100
- [ ] Use 5 use cases or less
- [ ] All use cases use Digital Financial Bank context
- [ ] At least 3 Mermaid diagrams included
- [ ] At least 3 comparison tables included
- [ ] Exam tips section includes service comparisons
- [ ] No repetitive content
- [ ] All sections within line budget
